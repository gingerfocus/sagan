* Problem 1
/Review the pipelined datapath shown on the last page and briefly explain the
role of each pipeline stage./

Stage 1 reads the instruction at PC out of memory and latches it. Stage 2 reads
data from the RF, prepares and immediate and parses what signals should be
active later in the seqence and latches it all. Stage 3 either preforms some
operation in the alu or memory and latches the result. Stage 4 write back any
data to the RF and possibly halts the machine. This is the stage that would
cause a pipeline hazard.
